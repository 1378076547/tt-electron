TTDesktop1.0 功能模块说明
=========================

一、程序定位
-----------
TTDesktop1.0 是一个基于 Electron 的 TT 工单辅助处理工具，核心目标：
1) 聚合待处理工单
2) 标准化处理流程（标题检测、优先级设置、话术发送）
3) 提升批量处理效率
4) 保持日志可追踪


二、模块总览
-----------

1. 自动处理模块（顶部控制区）
作用：按设定周期自动执行接单/处理流程。
包含功能：
- 处理人 MIS 设置
- 执行间隔设置（秒）
- 自动建群开关
- 自动发送话术开关
- 标题巡检（仅日志）开关
- 开始/停止按钮


2. 执行日志模块
作用：展示实时运行状态与结果，方便排查。
包含功能：
- 过程日志（成功/警告/错误）
- 打开日志目录
- 日志按天落地保存


3. 工单列表模块
作用：显示从 TT 同步的待处理工单并支持快速操作。
包含功能：
- 工单刷新（API + TT 页面并集，自动去重）
- 每条工单显示：创建时间、已历时、状态、优先级
- 排序：
  - 按优先级（高->低 / 低->高）
  - 按创建时间（新->旧 / 旧->新）
- 标题巡检（仅列表粗检）
- 标题检测（基于详情页规则生成建议标题）
- 当前工单优先级设置（非常紧急/紧急/高/中/低）
- 批量设置优先级、关键词自动升高(S3)
- 按地区拉 PM（需先选 PM 配置 CSV）
- 48h 时效提醒（见下文「十、工单历时与 48 小时时效提醒」）


4. 标题筛选与分类模块
作用：按标题快速定位特定类型工单。
包含功能：
- 标题关键词搜索（实时过滤）
- 分类筛选（并显示数量）：
  - 全部
  - 监控类
  - 打印类
  - 网络类
  - PDA类
  - 其他
- 关键词搜索 + 分类筛选可叠加


5. 标题规范模块
分为两个子模块：
- 标题巡检（轻量）
  仅基于列表标题做规范检查，不打开 TT 详情页。
- 标题检测（精确）
  打开工单详情页读取发起人架构路径、城市词典等信息，计算并给出/写入建议标题。
- 来单改标题（须先点顶部「开始」）
  范围：当前处理人 + 未处理/处理中/暂停（含他人转给你的单）。
  仅处理「开始」后新出现在列表中的工单（开始时已有单记入基线，不重复改）。
  工单面板按钮「来单改标题」：手动对当前新单检测，确认后写入 TT。
  顶部勾选「来单改标题（随开始）」：刷新列表发现新单时自动跑同一流程（仍须确认写入）。
  后缀规则：位置名仅 XX站 / XX仓 / XX店（无 XX库）；以仓结尾不补「站」；门店缺后缀补「店」；站类缺后缀补「站」。
  站/仓/店名：架构中带站/店/仓的实体段与「仓库/门店」字段结合判断；职能段（如门店运营、经营培训）不作网点名；字段不准时以字段+架构地区为准。
  解析失败（无架构/无法解析城市等）：跳过并写日志，系统通知提醒，不影响继续接单。


6. 优先级设置模块
作用：仅针对当前打开工单，自动改 TT 内优先级。
特点：
- 支持 S1~S5 文案映射（非常紧急/紧急/高/中/低）
- 兼容当前页面的优先级下拉展开机制
- 已增加失焦提交与延迟校验，减少误报


7. 话术模板模块
作用：根据工单标题关键词自动匹配发送话术。
包含功能：
- 中文默认话术（兜底）：未命中规则时，标题含中文优先用此项（须在界面填写并保存，无程序内置兜底）
- 英文默认话术（兜底）：未命中规则时，标题含英文优先用此项（同上）
- 多条关键词规则（关键词支持逗号、中文逗号、分号分隔）
- 命中优先级：从上到下；标题去空格后包含关键词即命中
- 保存后生效；可用「用当前工单标题试匹配」在执行日志预览
- 本地保存（localStorage）


8. 菜单栏模块
文件菜单：
- 显示窗口
- 打开 API 配置文件（tt-api.local.json，用于工单 API 拉单）
- 重启程序
- 退出

帮助菜单：
- 打开 TT 系统
- 开发者工具


9. 运行与分发模块
作用：保证可安装、可交付、可稳定运行。
现状：
- 已支持单实例运行（禁止重复开多个程序）
- 已支持 Windows 安装包（NSIS）
- 安装时支持选择安装目录（盘符）


三、典型使用流程
---------------
1) 启动程序并登录 TT（内置浏览器）
2) 在“工单”页点击“刷新”同步数据
3) 使用关键词/分类筛选目标工单
4) 按需执行：
   - 标题巡检（看问题分布）
   - 标题检测（规范化标题）
   - 设置优先级（当前工单）
5) 在“执行日志”查看结果与异常
6) 关注「已历时」与 48h 预警/超时提示，优先处理临近或已超时工单


十、工单历时与 48 小时时效提醒
------------------------------

【功能目的】
从工单创建时间（createdAt）起计算「至今已多久」，并在临近/超过 48 小时时主动提醒，
便于实时掌握处理时效，避免工单积压过久。


【一、已历时显示】
1. 位置：左侧「工单」列表，每条工单在「创建：xx」旁显示「已历时：xx」。
2. 计算方式：当前时间 − 工单创建时间（优先用 API 的 createdAt，并与页面抓取结果合并去重）。
3. 刷新频率：在「工单」页打开时约每 1 秒更新一次显示（无需手动刷新列表）。
4. 颜色含义（按已历时长短）：
   - 小于 4 小时：青色
   - 4～8 小时：黄色
   - 8～40 小时：橙色
   - 40 小时及以上：红色


【二、48 小时时效提醒规则】
1. 统计对象：状态为「未处理」「处理中」「暂停中」的工单（已关闭/已解决等不计入）。
2. 阈值（固定）：
   - 预警：创建满 40 小时（列表显示「即将48h」，距超时约剩 8 小时）
   - 超时：创建满 48 小时（列表显示「已超48h」）
3. 同一工单、同一阶段（预警或超时）在 30 分钟内只提醒一次，避免刷屏。


【三、提醒方式（四种同时可用）】
A. 列表内标签
   - 「即将48h」：已满 40 小时、未满 48 小时
   - 「已超48h」：已满 48 小时

B. 列表上方汇总条
   - 例如：「即将满 48h：2 单 · 已超时：1 单」

C. 执行日志
   - [48h预警] 工单标题 · 已历时 xx · 约 x 小时后满 48h
   - [48h超时] 工单标题 · 已历时 xx · 已超过 48 小时
   - 在「工单」页点击「刷新」后会扫描并可能写入日志；定时刷新历时不会重复打日志（有冷却）。

D. 顶栏与系统托盘
   - 顶栏「待处理」旁显示：预警 N · 超时 M
   - 托盘图标悬停同样显示 48h 预警/超时数量

E. Windows 桌面通知（可选）
   - 在工单工具栏勾选「Windows通知」后，达到预警/超时时弹出系统通知。
   - 需 Windows 允许本程序通知；默认关闭，按需开启。


【四、开关位置】
在「工单」页批量操作栏：
- 「48h时效提醒」：总开关，默认开启；关闭后仅保留「已历时」与颜色，不再预警/超时提醒。
- 「Windows通知」：默认关闭；开启后配合总开关使用桌面弹窗。


【五、使用建议】
1. 日常：保持「48h时效提醒」开启，优先处理带「已超48h」或红色历时的工单。
2. 离开工位：可额外开启「Windows通知」，避免漏看列表。
3. 刷新：点击「刷新」同步最新工单后，会重新扫描并可能产生新的日志/通知（仍受 30 分钟冷却限制）。
4. 若「已历时」显示异常，请先点「刷新」；创建时间以 TT/API 为准，列表格式为「05-29 14:35」时会按当前年份解析。


【六、与其它功能的关系】
- 与「隐藏已关闭」筛选一致：已关闭工单不参与 48h 提醒。
- 与自动接单/批量优先级无冲突；提醒仅展示与记录，不会自动改工单状态。
- 工单数据来源：API（tt-api.local.json 配置）+ TT 页面列表并集去重，避免重复计数。


十一、按地区拉 PM（CSV 与匹配说明）
--------------------------------

【CSV 格式】
表头：enabled,region_key,members_mis,note
members_mis 单元格内多人用英文分号 ; 分隔，每人建议写成：
  姓名/MIS
示例：
  张均杰/wb_zhangjunjie25;张铭曜/wb_zhangmingyao

【不必改表格列结构】
当前四列即可；关键是每个成员写成「姓名/MIS」，MIS 建议与 TT 大象里显示的一致（带 wb_ 更稳）。

【匹配规则（程序内）】
1. 优先：姓名 + MIS 都与搜索结果一致（MIS 支持 zhangjunjie25 与 wb_zhangjunjie25 等价）。
2. 若仅 1 条结果 MIS 相同、姓名略有差异（如多字少字），仍会选择并在日志中说明。
3. 搜索依次尝试：完整 MIS → 去 wb_ 的 MIS → 姓名 → 姓名/MIS。
4. 跳过时会写出「搜索列表」里可见的几条结果，便于对照 TT 修改 CSV。

【常见未选中原因】
- MIS 写错或与 TT 显示不一致（如少了 wb_ 且 TT 仅显示 wb_ 账号）。
- 姓名字形与 TT 不一致且 MIS 下有多人（无法唯一确定）。
- 该 MIS 在 TT 搜索无结果（离职、无权限、需用姓名搜索等）。

【no_confirm】
部分成员已勾选但弹窗「确定」未点到：日志会列已选人员；可手动点确定，或重试拉 PM。
